# Grill-me Skill

## Gate
Do not modify source code. Do not produce an implementation plan. Do not create files unless the user explicitly asks to save the clarification.

## Process
1. Restate the requirement in business language.
2. Identify business goal and success signal.
3. Extract constraints, impacted users, data entities, and risk areas.
4. Ask at most 5 critical questions.
5. State assumptions clearly so SPEC can proceed if the user accepts them.

## Output
1. **Current Understanding**
2. **Business Goal**
3. **Missing Information**
4. **Questions**
5. **Assumptions**
6. **Edge Cases**
7. **Acceptance Direction**
